About Flag Master

Flag Master makes it easy to hand content moderation of an ExpressionEngine site off to the users. With Flag Master you can create unlimited Profiles with custom options to allow site users the ability to mark, or "Flag", certain content based on your specific needs and custom criteria.

Flag Master works with both Channel Entries and Comments and can be set to auto close items based on flag count.

Installation

To install Flag Master you follow the same steps as any other ExpressionEngine add-on:

  1. Unzip the download
  2. Upload the extracted folder to your ExpressionEngine third party folder usually located at:
    /system/expressionengine/third_party/
  3. Navigate to your Add-Ons management page and choose Modules
  4. Select Flag Master from the list and click install. Choose to install all items.

Once the module package is installed you have to create your first Profile and create/assign Options to it.

Optionally, you can create a custom field within your channels using the Flag Master custom FieldType. This FieldType will display details about an entries current Flags and allow quick access to the module.

FieldType

Flag Master comes with a FieldType you can attach to your channel entries to view the Flags against your content. The FieldType lists the Flag Options selected and links to display details on those flags. It's recommended that the field is placed in it's own tab from the Publish Layout customization.

Profiles

Flag Master allows you to setup multiple, different, flag options for your content. You do this through Profiles. For example, you can setup a Profile for your Blog Comments and another for Blog Articles, each having it's own custom options to flag an item.

Add Profile

Simply click Add Profile and fill out the form. Note that the Type option is very important for Flag Master to relate data properly so check your work.

Flag Options

Once a Profile has been created you can add Flag Options. Here is where you setup the various options selectable for flagging content. You can setup options to be user defined (textarea option) as well as if a user has to be logged in to choose an option.

You can sort the Flag Options by drag and dropping the rows on the Profile View page.

Dashboard

The Dashboard is where you can view all the stats for your Flagged items. The Dashboard is broken up by either Entry or Comments.

View Flags

Clicking on a either a comment or entry from the Dashboard will take you to an overview of the flags for that item. This page displays the quick access to the item, dates for first and last flag, and a grouping of all flag options.

View Flag Option

To allow for detailed analysis of an individual comment or entrie's flags Flag Master stores a lot of details about each flag event.

Developers

Flag Master uses a prefix mechanism for all template tags. By default, the prefix is "fm" but you can change it by adding the variable "prefix=" to all tag pairs.

flag_master:simple_form

{exp:flag_master:simple_form}

This is the main template tag pair to use. It creates a form to create the Flag interface. It accepts the below parameters:

Parameters

profile_id (required)
The ID number for the profile you're using for this content. You can get this by viewing the Profile list page.

entry_id/comment_id (required)
Either an entry_id or a comment_id must be passed to this tag. Be sure to double check that your profile is setup for the content type you're using here.

return (optional)
The template path/group you want the user to return to after flagging an item. Note that it should NOT have a slash at the front. (eg: blog/my-blog-url-title/comments). If left blank the user is redirected to the home page.

prefix (optional)
The value to use for template tags within the tag pair. By default all variables are prefixed with "fm" but you can change that here.

Variables

Note! All examples below use "fm" as the variable prefix.

fm:type
The profile type

fm:auto_close_threshold
The amount of flags required to close an entry/comment.

fm:profile_name
The name of the profile being used.

fm:is_duplicate
Returns "1" if a user has already flagged an item under that profile. Useful for conditionals.

fm:options
Tag Pair to output the flag form fields.

fm:js_id
Used in between "fm:options" this will return the ID created for the form field option. Useful for creating JavaScript and field labels.

fm:radio_button
Used in between "fm:options" this will return the radio button to output for the option. Note that this is a plain radio button with no title.

fm:option_title
Used in between "fm:options" this will return the option's title.

fm:description
Used in between "fm:options" this will return the option's description.

fm:user_defined
Returns "1" if an options is set to be user defined. Useful for checking if a textarea is supposed to be output.

fm:textarea
Used in between "fm:options" this will return a textarea only if the option is set as User Defined.

Configuration

The date format you want Flag Master to use when handling dates. Note that the format should conform to the ExpressionEngine date format.
